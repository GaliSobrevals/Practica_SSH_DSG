Práctica SSH

Galilea Denova Sobrevals

a) ¿Qué tipo de servicio es SSH?

SSH es un servicio de conexión remota segura. Permite acceder a otro equipo a través de la red, pero de forma encriptada, evitando que alguien pueda ver o interceptar la información transmitida.

b) ¿De qué dos maneras podemos autenticarnos en un servidor SSH?

  • Contraseña: Es el método tradicional donde el usuario escribe manualmente una contraseña para acceder al servidor. Es fácil de usar, pero menos seguro porque las contraseñas pueden ser adivinadas, robadas o repetidas en varios sitios.
  • Llave pública y privada: Usa dos claves: una pública (que se guarda en el servidor) y una privada (que solo tiene el usuario). La conexión se valida automáticamente sin escribir contraseña y ofrece un nivel de seguridad mucho mayor, ya que está basada en criptografía y no en una clave que pueda ser adivinada..

c) ¿Qué es la autenticación interactiva mediante teclado?

Es cuando el servidor pide al usuario que escriba una contraseña (o varias preguntas de seguridad) durante la conexión SSH. Básicamente es el método tradicional donde tú escribes manualmente algo en el teclado para acceder.

d) ¿Qué diferencias hay entre ellas?

La autenticación por contraseña funciona pidiendo al usuario que escriba una contraseña cada vez que quiere conectarse al servidor. Este método es sencillo y no requiere configuraciones adicionales, pero es menos seguro. Las contraseñas pueden ser débiles, repetirse en varios servicios, ser robadas mediante ataques o incluso adivinadas por fuerza bruta. Además, dependen totalmente del usuario, lo que aumenta el riesgo de errores y fallos de seguridad.

En cambio, la autenticación mediante claves SSH utiliza un par de claves: una pública que se guarda en el servidor y una privada que solo posee el usuario. Cuando intentas conectarte, el servidor verifica que tu clave privada coincide con la pública, permitiendo el acceso sin necesidad de escribir una contraseña. Este método es mucho más seguro porque no puede ser adivinado y la clave privada nunca viaja por la red. También es más cómodo a largo plazo, ya que permite acceder rápidamente sin escribir contraseñas.

e) ¿Cuál es mejor? ¿Por qué?

La autenticación mediante claves SSH es considerada la mejor opción porque ofrece un nivel de seguridad mucho más alto que el uso de contraseñas. Las claves son más difíciles de vulnerar, no se pueden adivinar mediante fuerza bruta y permiten una conexión más rápida y cómoda al no requerir que el usuario escriba una contraseña cada vez. Además, la clave privada nunca se transmite por la red, lo que reduce significativamente el riesgo de ataques.

f) ¿Qué significan las siglas SSH?

SSH significa Secure Shell, que se traduce como "Consola Segura" o "Intérprete de órdenes seguro". Estas siglas reflejan el propósito principal del servicio: permitir una comunicación remota protegida mediante cifrado.

g) ¿Para qué puede ser usado SSH?

SSH puede ser utilizado para muchas tareas importantes, como acceder y administrar servidores de manera remota, ejecutar comandos, transferir archivos de forma segura usando SCP o SFTP, crear túneles cifrados para proteger conexiones y automatizar procesos que requieran conectarse a un servidor sin supervisión. Es una herramienta esencial para administración de sistemas y trabajo remoto seguro.
